# Known Problem
针对该项目，目前已发现如下问题，欢迎通过提交issue或者PR进行补充：

## 懒惰
在没有很好的提示词的情况下（哪怕有很好的提示此也会出现问题），模型在使用Stata-MCP的时候经常会懒惰，这表现为模型在执行任务的时候总是不去读log文件；当把读取log集成到stata-do后，模型新的懒惰方式是只写do-file而不去执行；当把三者集成在一起后模型会选择只给提供一段文本形式的代码而不去写入dofile。

## 蝴蝶效应
当出现一个小问题后，模型往往会因为一些问题而不知道如何解决这个问题，从而导致进入一个死循环，一直在尝试解决这个问题。比如在执行do-file的时候，如果有一个问题出错，可能是因为它并不是很熟悉Stata的语法导致的，它会不断尝试（但是是错误的方法）。
